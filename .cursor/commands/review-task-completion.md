Oceń ogólne wykonanie taska względem wstępnych założeń i intencji biznesowej. Działasz po implementacji i review.

KROKI:

1. odtwórz założenia z wątku: cel biznesowy, zakres z prepare-task / akceptacji implement-taks, co było out of scope
2. stan repo holistycznie: git status, git diff, git diff --cached, kluczowe pliki
3. odpowiedz: czy realizuje intencję (nie tylko literalne AC)? luki funkcjonalne mimo odhaczonych AC? scope nie wypłynął? spójność z codebase na merge?

WYNIK:

- jedno zdanie: "Wobec założeń taska: spełnione / częściowo / niespełnione"
- Ocena: 2–4 zdania — czy warto merge'ować
- Braki / ryzyka: konkretne luki (jeśli są)
- Poza scope: tylko jeśli coś powinno być osobnym taskiem

OUT OF SCOPE: pełny code review; naprawy; wymyślanie nowych AC.
